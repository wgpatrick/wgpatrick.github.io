Assumptions

  • Fed-batch monoclonal antibody (mAb) production process
  • Utilization: 70%
  • Total baseline COGS: $60/g
  • Raw Materials: $10.8/g (18%)
  • Labor: $9.0/g (15%)
  • QA/QC: $4.2/g (7%)
  • Utilities: $1.8/g (3%)
  • Facility Depreciation: $15.4/g (25.7%)
  • Equipment Depreciation: $12.0/g (20.0%)
  • Facility MRO: $6.8/g (11.3%)
  • Low-cost site saves 20% on labor, facility depreciation, and facility MRO; 10% on utilities. Equipment depreciation unaffected.
  • Facility cost slider only affects facility depreciation, not equipment.
  • MRO costs fall half as steeply with facility cost reduction.
  • Model based on: Monoclonal antibody production cost analysis

Training

Goal: Improve task performance such that autonomous operation is viable with teleoperation

Training approach:

1. Real-world data collection using tele-op on subset of tasks (~25)

2. Simulation data collection using simulated "digital twin" of operation

3. Test on real-world training set and 5-10 real-world test set

Questions:

1. How effective is real-world training using teleoperation? What is the rate of learning?

2. How effective is sim2real learning? What is the rate of learning (hours of data / failure rate improvement)?

3. How much transfer learning between tasks?

Once we better understand these questions, we can understand if this problem is tractable.

Simulation Key Assumptions

Training Structure

  • 1,000 total tasks across 10-15 unit operations
  • Real-world training: 25 tasks per 180-day campaign
  • Simulation training: 25 different tasks per day
  • Transfer learning: Improvements spread to untrained tasks

Learning Dynamics

  • Power law learning: Improvement ∝ hours-α
  • Initial failure rates: Log-normal distribution (~10% median)
  • GMP target: ≤0.01% failure rate
  • Learning floor: Cannot improve below 0.001%

Training Efficiency

  • Real-world: 24 training hours per task per day
  • Simulation: 10% efficiency vs real-world (adjustable)
  • Campaign cycles: New real-world tasks every 180 days
  • Transfer rate: 1-10% of direct improvements spread

Timeline & Scope

  • Total duration: 900 days (~5 real-world campaigns)
  • Success metric: All 1,000 tasks at GMP specification
  • Teleoperation backup: For tasks not yet autonomous

So, you're saying there's a chance?

Why is this possible?

  • Known, built environment
  • Discrete, known tasks
  • Tele-op should work for failing tasks
  • Can build "automation-friendly" systems to make hard tasks easier

Why not?

  • Tasks require too much dexterity
  • Too many difficult tasks to learn
  • Transfer learning doesn't work because tasks are highly varied
